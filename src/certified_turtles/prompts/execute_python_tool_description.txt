Не вызывай для просьбы только **показать/написать код** без запуска — тогда код в assistant_markdown, calls=[]. Серверное выполнение Python: запуск в песочнице (анализ, расчёты, симуляции, графики, HTTP). Полный скрипт в `code`; таймаут; импорты только из белого списка; запрещены open(), .open(), eval/exec. Таблицы: опционально передай `file_id` (как у workspace_file_path) — в скрипте будет константа CT_DATA_FILE_ABSPATH; второй файл — `file_id_2` → `CT_DATA_FILE_ABSPATH_2`. Затем import pandas as pd; df = pd.read_csv(CT_DATA_FILE_ABSPATH, encoding='utf-8', on_bad_lines='skip'). Живые HTTP-запросы (любой сервис): urllib.request / http.client или requests, вывод через print(). Если пользователь прислал в чате URL, API-ключ или токен — подставь их в код из текста сообщения (или собери заголовки из того, что он написал). Не подменяй это переменными окружения сервера: в контейнере свой ключ, у пользователя в сообщении — свой. Не подставляй плейсхолдеры вместо file_id. Либо сначала тул workspace_file_path и подставь absolute_path из JSON в read_csv. Если returncode не 0 — читай stderr, исправь код и вызови execute_python снова. Графики сохраняй в CT_RUN_OUTPUT_DIR (см. pathlib.Path(CT_RUN_OUTPUT_DIR) / 'plot.png'). Ответ тула: stdout/stderr, артефакты.
