# Open WebUI: ручной выбор моделей MWS GPT

Сборка включает **официальный образ** [Open WebUI](https://github.com/open-webui/open-webui). Автовыбор модели и отдельные «тулы» не настраиваются — пользователь сам выбирает модель из списка.

## Откуда берётся список моделей

1. Open WebUI запрашивает **`GET /v1/models`** у MWS GPT (тот же базовый URL, что и для чата).
2. Все записи из ответа попадают в **выпадающий список модели** в чате.
3. В `docker-compose` включено **`BYPASS_MODEL_ACCESS_CONTROL=True`**, чтобы обычные пользователи видели полный список из API, без дополнительной фильтрации по «workspace» в Open WebUI.

Фактический набор имён зависит от **allowlist вашего API-ключа** (если модели нет в листинге MWS для ключа, её не будет в UI).

## Как выбрать модель (UX)

1. Откройте **новый чат** или существующий диалог.
2. В **верхней части** окна чата найдите **селектор модели** (название текущей модели / «Select a model»).
3. Раскройте список и выберите нужную модель **вручную** — ответы пойдут в выбранную модель.
4. При необходимости обновите список: в интерфейсе Open WebUI в настройках подключений к провайдеру обычно есть действие обновления моделей (зависит от версии UI); при сомнениях перезапустите контейнер `open-webui`.

## Брендинг и язык

- **`WEBUI_NAME`** — имя в шапке (по умолчанию в compose: `GPTHub`).
- **`DEFAULT_LOCALE=ru-RU`** — локаль интерфейса по умолчанию.

## Если список пустой

- Проверьте **`MWS_API_KEY`** в `.env` и что `OPENAI_API_BASE_URL` указывает на `…/v1`.
- Выполните из **корня репозитория**: `uv run mws-gpt models` (тот же ключ). Не запускайте `python -m mws_gpt` из `src/…` — пакет называется `certified_turtles.mws_gpt`.
- При смене ключа или URL удалите том **`open-webui-data`** только если готовы потерять локальные чаты и настройки внутри контейнера.
