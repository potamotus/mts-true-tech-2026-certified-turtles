# Тулы GPTHub (Certified Turtles): что сделано и как улучшить

Документ описывает **все примитивные тулы**, зарегистрированные через `register_tool` (импорт из `src/certified_turtles/tools/builtins/__init__.py`), и **под-агентов** как вызываемые функции `agent_*` из `agents/registry.py`. Автовыбор модели и долговременная память здесь не рассматриваются.

---

## Примитивные тулы

### `web_search`

**Что делает.** Текстовый поиск в интернете через DuckDuckGo (`ddgs`): возвращает заголовки, URL и сниппеты. URL в запросе отклоняет и подсказывает `fetch_url`.

**Как улучшить.**

- Добавить **кэширование** похожих запросов (TTL), чтобы снизить лимиты и задержки.
- **Язык/регион** (параметр региона DDG, если доступен в библиотеке) для более релевантной выдачи.
- **Fallback**: при недоступности DDG — второй провайдер или явная ошибка с ретраем.
- Структурировать ответ: **дата индексации**, пометка «возможно устарело» для новостных запросов.
- Метрики: логировать `query`, latency, число результатов для отладки качества.

---

### `fetch_url`

**Что делает.** HTTP GET страницы, извлечение **plain text** (HTML→текст), лимит символов `max_chars` (500–20000).

**Как улучшить.**

- Для **JavaScript-heavy** сайтов — опционально headless-браузер (Playwright) как отдельный тул или флаг, с жёстким таймаутом и запретом по умолчанию.
- **Читабельный режим**: извлечение основной статьи (readability), а не всего body.
- **Robots.txt / rate limit** per host, чтобы не банили IP.
- Поддержка **редиректов**, выбор кодировки, `Content-Type` (не только HTML).
- Для PDF — отдельная ветка или отдельный тул `fetch_pdf_text`.

---

### `generate_image`

**Что делает.** Формирует URL генерации картинки через **Pollinations** (без ключа), параметры `width`/`height`, модели `flux` / `turbo`, возвращает JSON с `url` и готовым `markdown`.

**Как улучшить.**

- Добавить **проверенные пресеты** размеров и стиля (иконка, баннер, слайд 16:9).
- **Валидация промпта** (длина, фильтр небезопасного контента по политике продукта).
- Прокси через **MWS/другой провайдер**, если Pollinations нестабилен или блокируется.
- Опционально: сохранение PNG на диск в `GENERATED_FILES_DIR` и **стабильный URL** через ваш API, а не только внешняя ссылка.

---

### `generate_presentation`

**Что делает.** Сборка `.pptx` из JSON-описания слайдов на базе шаблона (`theme0.pptx`), типы слайдов: контент, секция, thanks, картинка по URL; файл отдаётся через `GET /files/...`.

**Как улучшить.**

- Больше **макетов** (две колонки, таблица, цитата), выбор темы из нескольких шаблонов.
- **Предпросмотр** / валидация слайдов до сборки (ошибки в одном слайде не ломают весь файл).
- Локальные картинки из `uploads` по `file_id`, не только `image_url`.
- Экспорт в **PDF** для шаринга без PowerPoint.

---

### `read_workspace_file`

**Что делает.** Читает файл из `UPLOADS_DIR` по `file_id`, UTF-8 с заменой ошибок, лимит ~512k символов, защита от path traversal.

**Как улучшить.**

- Явное определение **бинарных** форматов: не декодировать как UTF-8, а возвращать «используй execute_python / pandas».
- Параметры **offset/limit** или постраничное чтение для больших файлов.
- **MIME-тип** и размер в ответе; для `.csv` — первые N строк без полной загрузки в память.

---

### `workspace_file_path`

**Что делает.** Резолвит `file_id` в абсолютный путь, суффикс, подсказки для `execute_python`; отсекает плейсхолдеры.

**Как улучшить.**

- Возвращать **хеш/mtime** файла для кэширования решений модели.
- Поддержка **не только uploads**: сгенерированные файлы в другом корне, если появятся.
- Унифицировать сообщения с `execute_python`, чтобы модель реже путала `file_id` и путь.

---

### `execute_python`

**Что делает.** Запускает Python в отдельном процессе с **белым списком импортов**, запретом `open`/`.open()`, таймаутом; преамбула с `CT_RUN_OUTPUT_DIR`, привязка `file_id` → `CT_DATA_FILE_ABSPATH`, карта путей, хелпер `_ct_path` / `workspace_file_path` внутри скрипта; артефакты — URL `python_runs`. Для **публичных HTTP/JSON API** (без привязки к MWS в туле) разрешены `urllib`, `http.client`, `ssl` — модель может сгенерировать код запроса; пользователю отдаётся `stdout`/`stderr`.

**Как улучшить.**

- Явные **лимиты по CPU/RAM** (cgroups / `resource.setrlimit` где возможно).
- **Детерминированный seed** для воспроизводимых графиков (опция).
- Разрешить **безопасное** чтение только из `CT_DATA_FILE_ABSPATH` без полного запрета `open` — или оставить как есть и усилить документацию.
- Песочница: **отдельный пользователь** в Docker, read-only root кроме run dir.
- Логи: сохранять **полный скрипт** в артефакт при ошибке (опционально, с маскированием секретов).
- Опциональная защита от **SSRF** (блок частных IP для `urlopen`).

---

### `google_docs_read`

**Что делает.** Чтение Google Doc: публичный экспорт по ссылке без ключа пользователя; при настроенном service account — API для приватных доков, расшаренных на `client_email`.

**Как улучшить.**

- Кэш по `document_id` с коротким TTL.
- Явная **структура** (заголовки, параграфы) вместо сплошного текста, если нужно для агента.
- Обработка **очень больших** документов: чанки + `read_range`.

---

### `google_docs_append`

**Что делает.** Добавление текста в конец документа через Google Docs API при service account и правах редактора.

**Как улучшить.**

- **Идемпотентность** или `request_id` для защиты от дублей при повторном вызове тулов.
- Форматирование: не только plain text, но **базовые стили** (bold, списки) через API.
- Подтверждение **квот** Google и понятные ошибки при 403/429.

## Под-агенты (тулы `agent_<id>`)

Под-агент — это **не отдельный HTTP endpoint**, а вызов `agent_{id}` из родительского LLM: внутри `loop` запускается вложенный чат с урезанным набором примитивов (`SubAgentSpec.tool_names`).

### `agent_research`

**Что делает.** Тулы: `web_search`, `fetch_url`. Краткий ответ с источниками.

**Как улучшить.** Лимит шагов/токенов, дедупликация URL, явный шаблон цитирования (APA/короткие ссылки).

---

### `agent_writer`

**Что делает.** Только переписывание текста, **без** внешних тулов.

**Как улучшить.** Параметры тона/длины в схеме вызова `task`, стиль «по брендбуку» из system.

---

### `agent_deep_research`

**Что делает.** Многошаговое исследование (`web_search` + `fetch_url`), до 16 внутренних раундов, отчёт TL;DR / выводы / источники.

**Как улучшить.** Планирование под-вопросов как отдельная фаза, **оценка покрытия** источников, анти-шум (игнорировать мусорные домены).

---

### `agent_coder`

**Что делает.** Тулы: `execute_python`, `read_workspace_file`, `workspace_file_path` — код и данные.

**Как улучшить.** Явное разделение с `agent_data_analyst` по промптам; передача **ожидаемого формата вывода** (JSON schema) в задачу.

---

### `agent_data_analyst`

**Что делает.** Тулы: `workspace_file_path`, `execute_python`, `read_workspace_file`, акцент на CSV/XLSX и аналитике.

**Как улучшить.** Шаблоны pandas для типовых отчётов; автоматическая **проверка схемы** колонок перед агрегациями; визуализация по умолчанию (один график на метрику).

---

## Общие улучшения уровня платформы

- **Единый формат ошибок** JSON для всех тулов (`error`, `detail`, `hint`, `retryable`).
- **Трассировка**: `tool_call_id` / correlation id в логах от прокси до subprocess.
- **Тесты контрактов**: OpenAI schema для каждого тула совпадает с реальными обработчиками.
